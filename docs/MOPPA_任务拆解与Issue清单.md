# MOPPA 任务拆解与 Issue 清单（v1.0）

本清单基于 `docs/MOPPA_开发计划.md`，可直接用于创建 GitHub Issues（Epic + Story）。

## 1. 使用规则

- Issue 标题格式：`[M{阶段}][{模块}] {任务名}`
- 标签建议：`backend` `frontend` `data` `ops` `security` `priority:P0/P1/P2` `milestone:M0-M4`
- 优先级定义：
  - P0：阻断项，必须先完成
  - P1：主链路必需
  - P2：增强项

## 2. M0（第1周）安全与基建

## Epic M0-1：建立安全与开发基线

### Story M0-1.1（P0）统一配置管理模块
- 标题：`[M0][backend] 统一配置加载与校验`
- 目标：实现环境变量读取、默认值、启动时校验。
- 任务：
  - 创建 `config/settings.py`（或等效模块）
  - 区分 `dev/test/prod` 环境
  - 缺失关键配置时启动失败并给出错误信息
- 验收标准：
  - 本地 `.env` 可启动
  - 缺失关键项时报错明确
  - 文档提供配置项表格
- 依赖：无

### Story M0-1.2（P0）结构化日志与 trace_id
- 标题：`[M0][backend] 接入结构化日志与trace_id`
- 目标：请求级 trace_id 全链路可追踪。
- 任务：
  - 增加请求中间件生成/透传 trace_id
  - 日志输出统一字段：`timestamp level trace_id module message`
  - 错误日志包含异常类型和堆栈
- 验收标准：
  - 任意 API 请求可在日志中检索同一 trace_id
  - 500 错误具备完整错误上下文
- 依赖：M0-1.1

### Story M0-1.3（P1）错误追踪与告警通道
- 标题：`[M0][ops] 错误追踪与基础告警接入`
- 目标：关键异常可被主动发现。
- 任务：
  - 接入 Sentry（或预留抽象）
  - 配置告警路由（邮件/飞书/Slack）
  - 定义 P0/P1 错误级别
- 验收标准：
  - 制造一次异常可触发告警
  - 告警包含环境与 trace_id
- 依赖：M0-1.2

## 3. M1（第2-3周）数据层与核心模型

## Epic M1-1：数据库与迁移体系

### Story M1-1.1（P0）数据库连接与 Alembic 初始化
- 标题：`[M1][data] 初始化PostgreSQL与Alembic`
- 验收标准：
  - `alembic upgrade head` 成功
  - `alembic downgrade -1` 成功
- 依赖：M0-1.1

### Story M1-1.2（P0）核心表第一批建模
- 标题：`[M1][data] 建立event/question/task_execution表`
- 字段重点：`trace_id`、`status`、`idempotency_key`
- 验收标准：
  - 三表创建成功并有必要索引
  - `task_execution.idempotency_key` 唯一约束生效
- 依赖：M1-1.1

### Story M1-1.3（P1）核心表第二批建模
- 标题：`[M1][data] 建立prediction/ground_truth/score_record表`
- 验收标准：
  - 外键关系完整
  - 关键查询存在索引（`question_id`、`model_id`、`created_at`）
- 依赖：M1-1.2

## Epic M1-2：后端分层与基础 API

### Story M1-2.1（P1）后端目录分层
- 标题：`[M1][backend] 建立router-service-repository分层`
- 验收标准：
  - 新增业务接口按分层组织
  - 无业务逻辑直接堆在 router
- 依赖：M0-1.1

### Story M1-2.2（P1）核心实体 CRUD API
- 标题：`[M1][backend] 提供event/question/task_execution基础CRUD`
- 验收标准：
  - Swagger 可见接口
  - 至少覆盖 create/list/detail/update 状态
- 依赖：M1-1.2, M1-2.1

## 4. M2（第4-5周）流程引擎 MVP

## Epic M2-1：S1-S3 问题生产链路

### Story M2-1.1（P0）S1 事件采集与过滤
- 标题：`[M2][backend] 实现事件采集过滤与原因记录`
- 验收标准：
  - 过滤状态有 `passed/filtered`
  - 每条 filtered 记录至少一个原因标签
- 依赖：M1-2.2

### Story M2-1.2（P0）S2 自动出题
- 标题：`[M2][backend] 实现模板出题与重复检查`
- 验收标准：
  - 题目具备截止时间与验证条件
  - 7天窗口重复率可统计
- 依赖：M2-1.1

### Story M2-1.3（P1）S3 审核队列
- 标题：`[M2][backend] 实现问题审核状态流转`
- 验收标准：
  - 支持通过/驳回/修改
  - 驳回必须附原因标签
- 依赖：M2-1.2

## Epic M2-2：S4-S6 评测链路

### Story M2-2.1（P0）模型预测执行器
- 标题：`[M2][backend] 模型预测执行与结果落库`
- 验收标准：
  - 统一模型调用协议
  - 记录耗时、状态、错误信息
- 依赖：M1-1.3, M2-1.3

### Story M2-2.2（P0）真值回填任务
- 标题：`[M2][backend] 到期问题真值拉取与证据落库`
- 验收标准：
  - 到期扫描可执行
  - 真值包含证据链接与可信度

### Story M2-2.3（P0）评分引擎 v1
- 标题：`[M2][backend] 评分规则版本化与总分计算`
- 验收标准：

  - 同输入重复计算结果一致
- 依赖：M2-2.2

## Epic M2-3：调度与可靠性

### Story M2-3.1（P0）任务调度框架
- 标题：`[M2][backend] 主任务与补偿任务调度框架`
- 验收标准：
  - 支持每日主任务、每小时补偿任务
  - 执行记录写入 `task_execution`
- 依赖：M1-1.2

### Story M2-3.2（P0）幂等与重试
- 标题：`[M2][backend] 幂等键校验与指数退避重试`
- 验收标准：
  - 重复任务不重复写入
  - 支持 1/5/15 分钟退避策略
- 依赖：M2-3.1
