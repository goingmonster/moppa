# MOPPA 开发计划（v1.0）

## 1. 文档目标

本计划用于将当前仓库从“文档设计阶段”推进到“可运行 MVP + 可持续迭代”的工程状态，覆盖后端、前端、数据、调度、质量与运维基线。

## 2. 当前状态（基线）

- 已有：SOP、数据库与流程设计文档。
- 已有：FastAPI 与 Vue 脚手架可运行。
- 缺失：核心业务链路（S1-S7）代码实现。
- 缺失：数据库 migration、调度系统、评分与榜单 API、测试与 CI、监控告警。

## 3. 开发目标（8 周）

### 3.1 总目标

在 8 周内交付可上线灰度的 MOPPA MVP：

1. 打通 S1-S6 最小闭环（事件 -> 出题 -> 预测 -> 真值 -> 评分 -> 榜单）。
2. 提供基础运营能力（审核、任务监控、反馈闭环）。
3. 建立最低工程质量门禁（测试、CI、日志、告警、审计）。

### 3.2 验收定义（Definition of Done）

- 所有核心接口有 API 文档与示例。
- 主流程任务每日可稳定运行（失败可重试，可追踪）。
- 评分结果可复现（同输入同输出）。
- 周榜/月榜可自动生成并可查询。
- 关键路径有自动化测试与 CI 校验。

## 5. 工作分解结构（WBS）

### 5.1 后端

- API 分层：router/service/repository。
- 任务调度：主任务 + 补偿任务。
- 幂等机制：`task_type + business_id + date_window + model_id`。
- 评分引擎：规则版本化 + 可复现计算。

### 5.2 前端

- 模块 1：配置中心（数据源/模板/规则）。
- 模块 2：审核中心（问题列表 + 批量处理）。
- 模块 3：评测中心（任务状态 + 模型结果）。
- 模块 4：榜单中心（分榜、趋势、筛选）。

### 5.3 数据与运维

- 数据分层：原始事件层、业务实体层、统计快照层。
- 可观测性：日志、指标、告警三件套。
- 安全：RBAC、审计日志、密钥管理。

## 6. 风险与应对

- 外部数据质量波动 -> 增加过滤规则版本与抽样回看。
- 模型调用不稳定 -> 统一超时、降级与重试策略。
- 真值回填延迟 -> 引入延迟队列与超时告警。
- 评分争议 -> 评分规则版本快照 + 可复算接口。
