测试机数据管道设计流程（脱敏版）

一、数据管道系统（MOPPA）初始化准备
    1. 注册系统用户密码（权限为最高级别）
        1）username/password/role <ADMIN_USERNAME>/<ADMIN_PASSWORD>/<ADMIN_ROLE>
    2. 配置事件来源的数据源
        1）配置数据库源（测试/生产均禁止明文写入文档）
        地址: <PG_HOST>:<PG_PORT>
        schema: <PG_SCHEMA>
        数据库名称: <PG_DATABASE>
        用户名: <PG_USERNAME>
        密码: <PG_PASSWORD>
        2）以上配置通过环境变量或密钥管理服务注入
    3. 配置事件的过滤规则
        1）在界面配置规则
        2）配置规则等级，分别对应四种问题的规则等级，L1,L2,L3,L4
            L1：单选题
            L2：多选题
            L3：深度搜索层级（第3级）包含开放式事件
            L4：超级代理层级（第4级）涵盖高波动性、开放式事件
    4.配置问题模板
        1）对应不同规则等级的事件，生成模板
        2）模板需要和情报专家确认
        3）问题模板需要带有截止日期
    5.配置参与评测的模型：界面通过配置 OPENAI URL + API KEY 完成参与评测模型的配置（密钥仅引用，不展示明文）
    4.配置答案的评分规则

二、MOPPA自动化生成问题
    1.从数据源获取事件
    2.加载事件的规则
    3.通过AI使用规则，完成对每一个事件的过滤
    4.对过滤后的事件加载模板，AI自动提出模板问题
    5（可选），对L1和L2的问题，通过内置AI或者人工完成答案范围的穷举
    6.必要，人工筛选出高质量的问题，也可以对问题进行编辑

三、评测模型开始预测
    1. 加载问题
    2. 调用模型开始评测程序，并且按照约定好的预测结果格式，将数据入库，模型执行时长为30分钟，每天执行一次

四、 定时获取真实答案
    1. 根据问题配置好的答案发布时间，每天定时获取真实答案 调用真实答案预测系统
    2. 将真实答案更新到自动生成的问题中
    要求：必须保证答案的真实

五、 对不同模型的答案进行评分
    1.提取问题真实答案
    2.加载不同类型问题的评分规则
    3.给出评分（可以调用AI，也可以设计一套评分系统）
    4.对不同类型的问题进行加权计算给出一周或者一个月的总排行

六、 预测问题的社区功能
    1.对自动产生的问题开启人工评论和预测的功能
        1）评论交互
    2.展示不同模型对不同问题的预测结果
        1）人工可以评论
    3.……
